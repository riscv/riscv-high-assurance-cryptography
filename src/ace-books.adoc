== Structure of the ACE Specification: Separation of Architecture and Data Formats

The ACE ISA is split into three normative Books and a non-normative one.

* Book 1 defines the fixed part of the Unprivileged Architecture (see <<ACE-unpriv>>).
The instructions and architectural states are specified independently of the exact data formats they operate on.

* {empty}
+
--
Book 2 defines the Data Structures and Algorithms of the Unprivileged Architecture (see <<ACE-algorithms>>).

Within the RISC-V architecture, conformance to Book 1 requires full conformance to Book 2.
In other words, each non-custom algorithm implemented must exhibit the behavior fully specified in Book 2.

Book 2 is kept separate from Book 1 because the data formats used to manage keys are intended to be interoperable across different ISAs that implement ACE or equivalent functionality, and are therefore not specific to RISC-V. Those ISAs must support the format of the Metadata Header and the semantics of its fields verbatim as well.
--
+

* Book 3 defines the Privileged Architecture (see <<ACE-priv>>).

* Book 4 provides non-normative pseudocode examples illustrating the intended use of the algorithms defined in Book 2.
